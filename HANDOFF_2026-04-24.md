### 6.3 Tailwind vs spec §9 — **open, due before Phase 5**
Spec: "no custom CSS, even-toolkit only." Tailwind 4.2.2 is installed and used in the phone shell. Either:
- Rip out Tailwind, migrate phone UI to even-toolkit primitives.
- Amend spec to permit utility CSS alongside even-toolkit.

Call this before any phone UI work — otherwise we'll re-litigate mid-build.

### 6.4 *(New)* Minimap orientation model (Phase 4b) — **open**
Three candidates for the grid:
- **Cardinal ticks only** (N/S/E/W labels on canvas edges + tiny tick marks). Cheapest, matches "no grid, just reference" minimalism.
- **Full grid** (faint 4×2 grid lines behind the route). Easier to judge scale but visually noisy on 240×120.
- **Compass rose** (small N-pointing rose in a corner). Clear orientation, costs canvas real estate.

Recommendation: cardinal ticks + a single N arrow in the top-right corner. Lowest ink, unambiguous.

### 6.5 *(New)* POI image source (Phase 4c) — **open**
- **Wikipedia thumb endpoint** (`rest_v1/page/summary/<title>` returns a thumb URL): only works when POI has a wikiTitle. Source already being used for summary.
- **Wikimedia Commons search**: broader coverage but needs category matching — higher infra cost.
- **OSM tile crop as fallback**: shows location context rather than POI itself; misses user's "know what to look for" intent.

Recommendation: start with Wikipedia thumb for wiki-backed POIs, leave OSM-only POIs image-less (no placeholder — spec §9 frowns on decorative imagery).

### 6.6 *(New)* POI_LIST pagination UX (Phase 4d) — **open**
- **Infinite scroll** (cursor past last item loads next page): natural gesture, but wastes a tap.
- **"More" sentinel item** (`> Load more (21–40)`): explicit, one tap.
- **Refresh-on-walk** (detect ≥X meters moved since last fetch): zero-tap but opaque.

Recommendation: "More" item for page-through + a `> Refresh nearby` item at the bottom when user is geographically stale (≥200m). Both are explicit and debuggable.
