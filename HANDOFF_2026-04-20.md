## 2. Open Bugs (from 04-19 real HW + 04-20 simulator testing)

### A. POI_DETAIL line 2 truncation — **CONFIRMED both real HW + simulator**
Screenshot evidence: "★ landmark · 0.2 mi · ~4 min" gets cut off mid-line. Same on 576×288 glasses and in simulator frame.
**Root cause hypothesis:** 4 render rows (title / metadata / wiki body / action menu) are fighting for vertical space. Metadata row gets clipped.

### B. Action menu cursor coupled to wiki body scroll — **simulator confirmed**
- Long wiki article: user must scroll through all wiki text before cursor reaches action rows.
- Short wiki article: cursor scrolls straight into actions.
- On real HW (04-19): Read More / Navigate / Safari / Back all **non-responsive to tap + double-tap** entirely.
**User proposal:** Move actions to a separate screen *or* invoke them via long-touch gesture. Frees vertical real estate for line 2.

### C. Loading screen stuck — **real HW only**
User's phone location was unresolved; loading screen hung indefinitely with no recovery path. Copy is also too terse.
**User proposal:** Friendlier copy — *"Discovering interesting things near you…"* — plus a timeout/retry affordance.

### D. Auto-jump past POI list — **real HW only**
In Rego Park, Queens, user went straight into a single POI detail and never saw the list. Suspect reducer shortcuts when `pois.length === 1`, or there's a stale-state path that bypasses `POI_LIST`.
