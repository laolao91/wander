**`src/phone/App.tsx` ‚Äî wire the two stubs:**

```ts
case 'request-location':
  // Geolocation.getCurrentPosition ‚Üí dispatch location-acquired or location-failed
  navigator.geolocation.getCurrentPosition(
    (pos) => dispatch({ type: 'location-acquired', lat: pos.coords.latitude, lng: pos.coords.longitude }),
    (err) => dispatch({ type: 'location-failed', message: err.message }),
    { timeout: 10000, maximumAge: 30000 }
  )
  return

case 'fetch-nearby-pois':
  fetchPois({
    lat: effect.lat,
    lng: effect.lng,
    radiusMiles: effect.settings.radiusMiles,
    categories: categoryIdsToCategories(effect.settings.enabledCategories),
  }).then((page) => {
    dispatch({ type: 'nearby-pois-loaded', pois: page.items, fetchedAt: Date.now() })
  }).catch((err: unknown) => {
    dispatch({ type: 'nearby-fetch-failed', message: err instanceof Error ? err.message : 'fetch failed' })
  })
  return
```

**Boot ‚Äî load stale cache on mount:**

Before dispatching `nearby-refresh-requested` on first tab open, check:
```ts
const cached = await loadNearbyCache(kv)
if (cached) dispatch({ type: 'nearby-pois-loaded', pois: cached.pois, fetchedAt: cached.fetchedAt })
```
This lets the user see yesterday's results instantly before the live fetch
completes.

## 3. All remaining open issues

### 3.0 [HIGH] Field-test bugs ‚Äî 2026-04-27 (fix before or during Phase I)

Five issues surfaced from hardware testing this session. All are concrete
and actionable; none block Phase I from starting, but (a)‚Äì(d) should ship
in the same session as Phase I since they're all phone-side.

**(a) Settings tab ‚Äî full-width layout**  
The Settings content is pinned to a narrow left-aligned column instead of
filling the screen. Fix: audit `SettingsGroup` + `SettingsTab.tsx` for
any `max-w` or missing `w-full` class. The `SettingsGroup` wrapper likely
has a default max-width that needs overriding, or the outer `<div>` in
`SettingsTab` needs `w-full`. Same concern applies to `NearbyTab.tsx` when
it's built ‚Äî make sure content uses the full width from day one.

**(b) Settings tab ‚Äî category toggle tap target**  
Tapping the category label row toggles the setting (via `ListItem.onPress`),
but tapping the toggle pill itself doesn't register. Root cause: the `Toggle`
inside `ListItem.trailing` is likely getting its events swallowed by the
`ListItem` container. Fix options:
- Remove `onPress` from `ListItem` so the row isn't tappable; make the
  `Toggle` the only hit target (cleaner ‚Äî toggle pill is the expected UX).
- Or keep `onPress` but ensure the `Toggle` onChange propagates correctly
  (verify event bubbling from even-toolkit's Toggle implementation).
Recommended: drop `ListItem.onPress`, toggle only via the pill.

**(c) Settings tab ‚Äî no way to push changes to glasses**  
After toggling categories or moving the slider there's no button to
trigger a glasses refresh. The sync info card says "Tap Refresh on the
Nearby tab" but users shouldn't have to context-switch to trigger it.
Fix: add a **"Send to glasses"** (or "Refresh Nearby") `Button` at the
bottom of the Settings tab. On tap it dispatches `nearby-refresh-requested`
which (once Phase I is wired) kicks the full locate‚Üífetch cycle.
The button should be disabled / show a spinner while `syncStatus === 'syncing'`.

**(d) Glasses ‚Äî list view missing distance**  
POI rows in the glasses list show name only. The spec and user expectation
is `Name... ¬∑ 0.30 mi`. The `Poi` object already has `distanceMiles`; it
just isn't being used in `renderPoiList`.  
Fix: in `src/glasses/render.ts`, update the list row format from:
```
‚òÖ Central Park...
```
to:
```
‚òÖ Central Park... ¬∑ 0.30 mi
```
Truncate the name to fit; the distance label is short and should always show.
File: `src/glasses/render.ts` ‚Üí `renderPoiList` ‚Üí the per-POI line builder.

**(e) Glasses ‚Äî loading / title screen WANDER not centred**  
Carried from ¬ß3.5 below. Now confirmed on real hardware from this session's
screenshot. `CHARS_PER_LINE = 65` in `render.ts` assumes monospace; G2
font isn't. WANDER appears left of centre and the rule line extends too far.
Fix: empirically tune `CHARS_PER_LINE` downward (try 48‚Äì52) until centred,
or switch the LOADING screen to a dedicated render path with hard-coded
padding instead of the generic `centeredLine()` helper.
File: `src/glasses/render.ts` ‚Äî `CHARS_PER_LINE` constant + `renderLoading`.
