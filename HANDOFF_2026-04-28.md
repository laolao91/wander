## 1. What shipped this session

### `7ad633b` — Phase I + layout redesign + WANDER centering

**`src/phone/App.tsx`** (major rewrite):
- Replaced `AppShell + NavBar (top)` with explicit `flex flex-col h-screen w-full`
- Bottom tab bar matching mockup `p-tabbar`: Nearby (◉) | Settings (⚙) with icons
- Custom header: Wander logo pill (W dot) + current tab name
- `request-location` wired: `navigator.geolocation.getCurrentPosition` → `location-acquired` / `location-failed`
- `fetch-nearby-pois` wired: `fetchPois()` → `nearby-pois-loaded` / `nearby-fetch-failed`
- Boot loads stale nearby cache: `loadNearbyCache(kv)` → `nearby-pois-loaded` if cached (user sees yesterday's results instantly)
- NearbyTab auto-dispatches `nearby-refresh-requested` on mount if `fetchStatus === 'idle'`

**`src/phone/tabs/NearbyTab.tsx`** (new):
- `RefreshBar`: "Updated X min ago · N places found · ↺ Refresh" — tap → `nearby-refresh-requested`
- POI list grouped by API category using `SectionHeader` + `ListItem` with distance subtitle
- Inline loading spinner when refreshing stale data
- Error banner (stale data still visible on network error)
- Full-screen loading state for first load
- Full-screen error states with retry action button for `error-location` and `error-network`
- Empty state ("Nothing nearby") for `success` with zero results
- Auto-refresh on mount: `useEffect` dispatches `nearby-refresh-requested` when `fetchStatus === 'idle'`
- Location label shown in App header when `nearby.location` is set

**`src/glasses/render.ts`** (WANDER centering):
- Added `LOADING_CHARS_PER_LINE = 120` separate from `CHARS_PER_LINE = 50`
- Root cause: space glyphs on G2 proportional font are much narrower than letter glyphs → `centeredBlock` at width=50 adds too few spaces, WANDER drifts left
- `centeredBlock` now accepts an optional `width` param; LOADING passes 120 for more left padding
- **Hardware verify needed:** if WANDER still drifts left, increase toward 140; if overcorrects right, reduce toward 100
