## 5. Open Architectural Decisions

### 5.1 ~~Action menu design (2a/2b/2c)~~ — **CLOSED 04-20**
2a shipped. POI_ACTIONS screen lives on its own; POI_DETAIL is read-only.

### 5.2 Dispatch bus pattern (phone ↔ glasses reducer) — **open, due before Phase 5**
Two candidates:
- **Module singleton** — simplest, one import, but couples tests to import order.
- **Event emitter** — cleaner test isolation, more ceremony.

Recommendation: module singleton unless Phase 5 uncovers a test-isolation pain point.

### 5.3 Tailwind vs spec §9 — **open, due before Phase 5**
Spec says "no custom CSS, even-toolkit only." Tailwind 4.2.2 is installed and used in the phone UI shell. Two resolutions:
- Rip out Tailwind, migrate phone UI to even-toolkit primitives.
- Amend the spec to permit utility CSS alongside even-toolkit components.

Needs a call before phone Settings/Nearby work ships, or we'll re-litigate mid-session.
